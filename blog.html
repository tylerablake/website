Creating Profiles with .NET MVC!

by Tyler Blake

Posted on March 24, 2015 at 7:00 PM

How to create profiles in .NET MVC!

1) Create the framework

I initated the project using the built in MVC .NET Framework with Bootsrap through Visual Studio.

2) Create Views

Then I created a view folder for Profiles, and inside of the folder I created an index view (for the initial landing page when a user clicks "Profile" in the nav bar), an edit view (for when the user wants to edit the profile, and a delete view (for when the user wants to delete the profile)..

3) Create the Model

Then I created a model which is a class that houses all the properties for the profile object.

4) Create Controllers

Then I created a controller using the sacafolding properties of Visual Studio for the profile views. In the controller I created methods that return views, as well as return data from the model.

5) Load Data

Then I created a controller to pull in the data from the Model, this way the page would be initiated with data that could then be edited or deleted by the user.

6) Test

Lasty, I tested to ensure all data was in the model, being accessed by the controller, and being sent to the view to be displayed to the user. As well as formatting the page to look more like a profile.

Getting Started with AngularJS

by Tyler Blake

How to foreach from a JSON array using Angular!

1) Initialization (ng-app)

The ng-app command is used as an attribute on an element to relay to the angular.js file which part of the webpage to interact with.

2) Data

An array of objects is stored in the tutorials.json file to be accessed by angular.

3) Access (get)

The get command allows angular to connect with the .json file in order to access the stored object array.

4) Step through objects (ng-repeat)

The ng-repeat class allows angular to step through an array of objects printing them to the page.

5) Page layout (ng-route)

The ng-route command enables navigation through the webpage, this way you only need to include the heading and navbar 1 time through all of your different pages.

Intro to Node JS

by Tyler Blake

Posted on Feb 24, 2015 at 6:00 PM

What is node js?

Nodejs is a cross platform runtime environment for networking and other server side applications. It was created by Ryan Dahl and sponsored by Joyent.

Node js utilizes a dynamic language in Javascript on a VM and uses V8 making it very powerful. With that, it is able to handle large amounts of connections in a single process due to its non-blocking IO.

Note: Node js is NOT a javascript library!


What makes node different than developing with more traditional server side languages?

Node doesn't halt, it only idles when needed. This allows the server to handle more connections and concurrency. It does this instead of sleeping, because when sleeping it can't receive more requests. This creates It knows to end execution if it is no longer needed.

It utilizes V8(a javascript engine running in Chrome) to aid in debugging, similar to Chrome Development Tools where you can place breakpoints and step through code. Although it is not as good as Chrome's Dev Tools it is still a nice feature to have.


What is the concept of non-blocking IO?

Non-blocking IO allows you to run multiple tasks in a single process, instead of opening a thread for each process you want your application to do.

This is a better alternative because opening multiple tasks or processes could lead to bottlenecking and is sure to end up leading to undesired inefficiencies in your application.
